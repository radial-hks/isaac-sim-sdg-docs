## 三阶段学习路径

### Phase 1：建立心智模型（1-2 周）

目标：能启动 Isaac Sim，写一个 Standalone Python 脚本创建物理场景，理解 Core API 封装关系。

```
01_Concepts → "Isaac Sim 是什么" + 架构分层图
    ↓
02_Development_Fundamentals → Python Scripting Concepts（Standalone vs Interactive）
    ↓
    Core API Tutorials 1-2（Hello World / Hello Robot）
    ↓
    Environment Setup、Robots Simulation
```

Phase 1 结束时你应该能：
- 用 `SimulationApp({"headless": True})` + Standalone Python 创建 Cube/GroundPlane
- 理解 `RigidPrim` / `Articulation` 包装了什么
- 知道 USD Stage ↔ UE5 Level 的对应关系

### Phase 2：开发层进阶（2-4 周）

目标：能写一个完整的 Extension（自定义 OmniGraph 节点），用 Replicator 做 Domain Randomization。

```
20_OmniGraph → OGN 节点 + Python Scripting
    ↓
03_Extension_Development → 从零搭建自定义 Extension
    ↓
11_Replicator_Core → Getting Started + Modular Scripting
    ↓
14_Domain_Randomization → Camera/Light/Geometry/Transformation randomizers
```

Phase 2 结束时你应该能：
- 写出自定义 OGN 节点并注册到 ActionGraph
- 用 Replicator API 对场景的 pose/light/material 做随机化
- 理解 Writer 输出 COCO/YOLO 标注的流程

### Phase 3：SDG 工程化（4-8 周）

目标：搭建可重复运行的 Headless SDG pipeline，输出高质量训练数据。

```
16_SimReady_Assets → 在 Nucleus 上找 SimReady 资产 + 物理材质
    ↓
15_Sensors_and_Annotators → 相机配置 + RTX Annotators（seg/BBox/depth）
    ↓
12_SDG_Workflows → Object-Based / Scene-Based SDG 案例
    ↓
04_Headless_and_Deploy → 容器化运行、远程可视化
    ↓
17_SDG_Pipeline → Cosmos + InfiniGen + Metropolis Pipeline
```

Phase 3 结束时你应该能：
- 在 Docker 容器里 headless 跑完一次 1000 帧的 SDG 任务
- 输出 COCO 格式标注 + RGB + depth 三模态数据
- 知道如何验证合成数据质量（分布可视化、标注抽检）

## UE5 → Isaac Sim 概念速查

| UE5 概念 | Isaac Sim / OpenUSD 对应 | 差异要点 |
|----------|-------------------------|---------|
| Level | Stage | USD Stage 是声明式数据描述，不是运行时对象 |
| Actor | Prim（Xform 类型） | Prim 更底层，可以是几何/材质/任意 Schema |
| Component | Prim 子节点 + Schema API | 无 Component 概念，用 `Articulation`/`RigidPrim` 等 View 类包装 |
| Blueprint | OmniGraph + Extension | OmniGraph 是逻辑编排，Extension 是打包分发 |
| Material | MDL + PhysX RigidBodyMaterial | MDL 视觉 + 物理材质分离 |
| Widget / UI | Carb Settings / GUI Extension | 用 Kit UI 框架 |
| World Partition | USD Layer | 分层加载，Composition Arc 机制 |
| Tick / Event | OmniGraph OnPlaybackTick / Python Callback | 两种机制并存 |
| Gameplay Ability / Behavior Tree | Behavior Tree Gen（SDG 模块）| Isaac Sim 有内置行为树生成器 |
| NavMesh | Occupancy Map Generator | 数字孪生导航图 |
| Datasmith | USD Import / URDF / MJCF | 资产导入走 OpenUSD 标准格式 |
| Niagara | Replicator + Particle System | 用于 SDG 场景中的特效/粒子随机化 |
| Landscape | Height Map + Terrain Extension | 地形导入 |
| LOD | USD Variants + Level of Detail | 用变体集管理 |
| Replication (Net) | ROS 2 Bridge / UCX | 物理仿真的同步机制 |

⚠️ **核心范式差异**：UE5 是有状态的 OOP 运行时（GameMode、Actor tick、GC），OpenUSD 是**声明式场景描述**，Stage 上的 Prim 修改通过 Schema API 直接写入数据层。**不要用 UE5 的 OOP 心态写 USD 代码。**
